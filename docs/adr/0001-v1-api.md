# ADR 0001: v1 API boundary

- Status: Accepted for staged implementation
- Date: 2026-07-18
- Baseline: `d779a8165638da22d7c10b149e04ac637b9603cf`
- Decision owners: project maintainers

## Context

The pre-v1 root package combines ordinary typed JSON, three document models,
streaming, transforms, native code-generation hooks, trusted output, and a
large low-level SIMD surface. Each family has a defensible use case, but their
ownership and compatibility promises differ enough that one stable package is
not a useful boundary.

The v1 boundary must make the common path obvious, reduce the audit perimeter,
and keep specialized facilities available without promising that experimental
compiler or unsafe details will remain stable. This ADR decides the target
surface before code moves begin. It does not authorize a flag-day rewrite.

## Decision

The module will expose three stability levels:

1. The root package is the stable interface for typed JSON, validation,
   standard-library-style transforms, and streams.
2. `document` is the advanced interface for borrowed or owning navigation,
   indexes, pointers, and forward cursors.
3. `experimental/codec` is the unstable interface for generated-code hooks,
   native cursors, and explicitly trusted output.

Low-level kernels are implementation details by default. The `simd` package is
not part of the v1 compatibility promise until a later ADR identifies a small,
checked facade with real external users. Unchecked and pointer-oriented kernels
move under `internal` rather than being re-exported from an experimental API.

Implementation note (2026-07-19): the pre-v1 Stage 1 and Stage 2 structural
API moved from `simd` to `internal/kernels` without compatibility wrappers.
The precondition-based string and UTF-8 scanner implementation then moved to
`internal/scanner`; root hot paths call it directly. The checked public scanner
wrappers were subsequently removed rather than preserve a facade over those
implementation entry points. The pre-v1 `simd` package retains numeric and time
helpers plus effective backend reporting while their eventual facade is
decided. Its duplicate public CPU-capability mirror was removed; raw capability
detection and routing remain owned by `internal/scanner`.

The `document` migration began with the dependency-free `Kind`, `IndexOptions`,
and `PointerError` vocabulary on 2026-07-20. The deprecated root `Kind` alias
and constants were removed before v1; public kind-returning methods now expose
`document.Kind` directly. The root `IndexOptions` and `PointerError` aliases
have also been removed. The staged root `BuildIndexOptions` entry point accepts
`document.IndexOptions` directly until the index implementation moves, and all
pointer parsers and selectors return `*document.PointerError` directly.

The index storage and address-space sentinels moved to `document` on
2026-07-20. Root index entry points return those document-owned values during
the staged model migration; no root forwarding variables preserve the old
selectors.

## Stable root package

The following table records every retained root symbol family. Symbols in one
row share the stated ownership and concurrency contract.

| Symbols | Primary use | Why it is distinct | Ownership and concurrency | v1 promise | Required performance evidence |
| --- | --- | --- | --- | --- | --- |
| `Marshal`, `Unmarshal` | Occasional typed encode/decode | Lowest-friction equivalent of ordinary `encoding/json` calls | Inputs remain caller-owned; owned decode is the default; package caches are concurrent-safe | Standard-library compatibility where documented; error differences remain typed and documented | High-level typed corpus, first-call, and allocation benchmarks |
| `CompileEncoder`, `Encoder`, `EncoderOptions`, `EncodeError` | Repeated typed encoding and caller-buffer reuse | `Marshal` cannot expose compiled-plan reuse or append semantics | Encoder plans are immutable and concurrent-safe; source and destination buffers remain caller-owned | Stable options and output/error semantics after v1 | Compiled-reuse corpus, tiny encode, hooks-disabled control, compile cost, binary size |
| `CompileDecoder`, `Decoder`, `DecoderOptions`, `DecodeError`, `UnsupportedTypeError` | Repeated typed decoding, destination reuse, and explicit ownership options | `Unmarshal` cannot express zero-copy, replacement, or field-selection policy | Decoder plans are immutable and concurrent-safe; destination mutation is caller-serialized; zero-copy aliases are explicit | Stable compatibility, ownership, merge/replace, and error-path semantics after v1 | Owned/zero-copy corpus, tiny decode, compile cost, destination-reuse allocations |
| `Valid`, `Validate`, `SyntaxError` | Strict validation of exactly one JSON document | Avoids materializing typed or document values | Source is borrowed for the call only; functions are concurrent-safe | Strict grammar, depth, and error-offset contract | Validation corpus, malformed near-valid corpus, portable/SIMD parity |
| `Compact`, `AppendCompact`, `Indent`, `AppendIndent`, `Canonicalize`, `AppendCanonicalize` | Standard-library-style document transforms | Encoding a Go value does not preserve source spellings or provide canonical member ordering | Source and destination follow ordinary slice aliasing rules; functions have no shared mutable state | Exact validation and formatting semantics are stable | Transform corpus and caller-buffer allocation benchmarks |
| `Reader`, `ReaderOptions`, `NewReader`, `NewReaderWithOptions`, `DecodeFrom`, `DecodeNext`, `ErrReaderClosed` | Framed JSON input and typed stream decoding | A stream owns refill, framing, offsets, and value-lifetime state that one-shot decode cannot provide | Reader owns a rolling buffer, is single-goroutine, and invalidates borrowed data on advance; compiled decoders remain concurrent-safe | Framing, limit, offset, close, and invalidation behavior are stable | Fragmented reader, huge-value limit, tiny framed decode, retained-buffer benchmarks |
| `Writer`, `NewWriter`, `EncodeTo` | Valid token or typed streaming output | A stream must preserve container and separator state across calls | Writer is single-goroutine and owns its buffer; compiled encoders remain concurrent-safe | Token-state, flush, error, and separator behavior are stable | Tiny writer, buffered encode, flush/error benchmarks |

Exported methods on retained types are retained when they implement the primary
use in the table. Before v1, every such method must document semantics,
ownership, concurrency, errors, and any allocation guarantee.

### Root symbols that do not graduate unchanged

The provisional `ValidNumber`, `ValidateNumber`, `ValidString`,
`ValidateString`, and `ValidateOptions` entry points were removed before v1.
Private equivalents retain the parser's exact scalar checks, error offsets,
and depth-aware validation oracle without exposing a second validation API
family. The stable root contract is `Valid`, `Validate`, and `SyntaxError`.

The provisional `Value.AppendIndent` method was also removed before v1. The
package-level `Indent` and `AppendIndent` functions are the single formatting
contract; keeping a second tree-walking implementation would duplicate both
behavior and maintenance work.

## Advanced `document` package

The document package owns all persistent or borrowed navigation models so that
their differences can be explained together instead of competing with typed
decode in the root.

| Target family | Current symbols | Primary use and uniqueness | Ownership and concurrency | Compatibility and evidence |
| --- | --- | --- | --- | --- |
| Borrowed raw selection | `RawValue`, `GetRaw*`, `EachArray*`, `EachObject*` | One or a few zero-materialization selections; unlike an index it does not retain navigation storage | Values alias immutable source; concurrent reads are safe while source remains live and unchanged | Stable in `document` after duplicate-key and partial-validation contracts are resolved; pointer and corpus benchmarks justify retention |
| Owning ordered tree | `Value`, `Member`, `Kind`, `Parse*`, `Options` | Retained ordered navigation independent of caller input; unlike typed decode it preserves a document model | Default parse owns backing; explicit zero-copy mode aliases immutable input; immutable values are concurrent-readable | Stable in `document`; owning/zero-copy behavior has differential and lifetime tests |
| Caller-backed index | `Index`, `Node`, `IndexEntry`, `IndexOptions`, `BuildIndex*`, `RequiredIndexEntries`, `ErrIndexFull`, `ErrIndexTooLarge` | Repeated out-of-order navigation without building an owning tree | Index aliases immutable source and caller storage; immutable reads are concurrent-safe | Stable in `document`; index corpus, storage exhaustion, address-width, and retained-memory evidence required |
| Pointer selection | `CompiledPointer`, `CompilePointer`, `MustCompilePointer`, `PointerError` | Reusable RFC 6901 parsing shared by every document model | Compiled values own their tokens and are immutable/concurrent-safe | Stable RFC 6901 and duplicate-key semantics; compiled/uncompiled parity required |
| Navigation cursors and iterators | `ArrayIter`, `ObjectIter`, `FieldCursor`, `ValueFieldCursor`, `ValueCursor` | Forward traversal where retaining a full index or tree is unnecessary | Cursors borrow their parent model or Reader window and are single-owner values; parent invalidation applies | Conventional `Next`/`NextKind`/`NextRaw` iteration graduates; forced-route and lifetime tests remain required |

`ScanFirstRaw*` remains provisional because its early-exit validation and
first-duplicate semantics differ from fully validating, last-duplicate
selection. It must either gain a name and package location that make that
contract unmistakable or be removed.

## Experimental generated-code API

The following symbols move to `experimental/codec` and carry no v1 source or
behavior compatibility promise:

- `MarshalerSimd` and `UnmarshalerSimd`;
- `DecodeCursor`;
- `Field`, `FieldSet`, `MakeField`, and `MakeFieldSet`;
- `TrustedAppender`;
- the `,inline` extension.

The package must separate a checked token-writer interface from an explicitly
trusted interface. The word `Trusted` must appear at the interface boundary,
not only on a value passed several calls later. Checked output cannot silently
corrupt its enclosing document. Trusted output remains opt-in, is intended for
generated code, and must have validation-tag tests.

`DecodeCursor` is passed by value; a retained copy keeps its input alive but is
disconnected from the enclosing decode. `TrustedAppender` is call-scoped and
must not be retained. Neither value is concurrent. Plans that dispatch hooks
remain immutable and concurrent-safe. Every unsafe or allocation claim
requires hook-integrity, lifetime, checkptr, and end-to-end benchmark evidence.

## Removal candidates

Removal is the default unless the listed evidence is produced before the move
that would otherwise delete the symbol.

| Candidate | Reason | Evidence required to retain |
| --- | --- | --- |
| `Codec`, `CodecOptions`, `CompileCodec` | Removed pre-v1: the wrapper forwarded to Encoder and Decoder and added a second adaptive hint | No external user evidence or distinct end-to-end win was found; retained Encoder, Decoder, EncodeTo, and DecodeFrom cover its contracts |
| `EncoderOptions.UnsortedInlineFields` | Removed pre-v1: it added an ordering knob to a nonstandard extension and kept an alternate executor branch | No multi-architecture workload demonstrated a material win; deterministic sorted output remains |
| `NewReaderSize`, `Reader.SetMaxValueBytes`, `ErrReaderStarted` | Removed pre-v1: duplicate construction and mutable configuration paths | `NewReaderWithOptions` preserves buffer sizing and value limits without a mutable started state; no distinct workload or ergonomics case was found |
| `NewWriterSize` | Removed pre-v1: the ambiguous size parameter coupled eager allocation to a completed-value flush threshold without limiting value size or retained capacity | `NewWriter` preserves the documented streaming path and explicit `Flush` controls latency; repository tests use a private threshold constructor, and no external caller or distinct workload justified a public tuning contract |
| `FlatArrayIter`, `FlatObjectIter` | Removed pre-v1: shape-specific duplicates of the general iterators | No repeatable end-to-end gain was recorded on both ARM64 and AMD64; the general iterators retain the conventional `Next` family |
| Iterator `Valid`/`Current`/`Advance` forms | Removed pre-v1: duplicate by-value loop shape beside conventional `Next` | Native run `29729155804` found Advance 39–49% faster on ARM64 but 8–33% slower on amd64 (all p=0.000, 0 allocs), so it failed the required cross-architecture win |
| Root aliases for document and hook symbols | Root `Kind` aliases were removed pre-v1; future migration aliases would preserve the oversized root indefinitely | Temporary deprecation bridge only, with a published removal release |
| Checked `simd` scanner facade | Removed pre-v1: sixteen wrappers duplicated the internal scanner boundary without a root production caller or distinct public contract | Root calls `internal/scanner` directly; scanner parity remains owned by that package |
| `Parse16Digits`, `Parse16DigitsChecked`, and their `Info` fields | Removed pre-v1: the public parsers had no root production caller, duplicated the root's inlinable two-word reduction, and exposed parse-backend policy that always reported scalar | `All16Digits` and `Parse8Digits` retain the reusable classification and reduction primitives; root owns its inlinable sixteen-digit combination |
| Public `CPUFeature`/`CPUFeatures` mirror and `Info.Features` | Removed pre-v1: it duplicated internal detection and exposed reporting-only capabilities beside the effective backend result | `Current` retains the selected backend names and widths; native internal tests cover detection and routing policy |
| Scalar and options validator facade | Removed pre-v1: five entry points duplicated the stable whole-document validation family without external user evidence | Private validators retain number/string corpus coverage, exact errors, and depth-aware route oracles; public callers use `Valid` or `Validate` |
| Public unchecked SIMD and stage kernels | Pointer and capacity preconditions are not a stable user-level contract | None for v1; move under `internal` |

## Migration sequence

1. Add external-package contract tests for the retained root surface.
2. Create dependency-neutral internal implementations so `document` does not
   import the root package and cause cycles.
3. Introduce `document`, move one model at a time, and provide deprecated root
   forwarding functions or aliases for one documented pre-v1 migration window.
4. Introduce `experimental/codec`, move native hooks, and keep an explicit
   validation build mode.
5. Measure removal candidates on ARM64 and AMD64, then delete or retain them in
   separate commits.
6. Remove migration aliases before the v1 tag; do not make deprecated aliases
   part of the v1 compatibility promise.

Every step must build independently. A public behavior change requires a
follow-up ADR and differential external-package tests. A hot-path move requires
paired performance gates against the preceding green commit as well as the
fixed [maintenance baseline](../maintenance-baseline.json).

## Release conditions

The stable label takes effect only when all of the following are true:

- the maintainer has selected a project license and root release notices are
  present;
- the portable stable-Go build works, or an exact blocker ADR is accepted;
- ownership, concurrency, limits, and errors are documented for every retained
  symbol;
- root aliases for document and experimental APIs are removed;
- ARM64 and AMD64 correctness and performance gates are green;
- no maintained latency row exceeds its gate, allocations/op do not increase,
  and repeatable B/op growth is resolved or explicitly approved.

## Consequences

Users get one obvious package for ordinary JSON, one package for advanced
documents, and an honest unstable boundary for generated-code machinery.
Moving symbols creates pre-v1 migration work and temporarily increases wrapper
code. That cost is bounded by the migration window and is preferable to making
three ownership models and unchecked output permanent root commitments.

This ADR does not choose a project license, add a public feature, or authorize
new specialization work.
